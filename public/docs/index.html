Introduction

Base URL: https://rrn24.techchantier.com/tutor-finder/public/api

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Authenticating requests

This API is not authenticated.

Application Management

APIs for managing gig applications

List Applications

requires authentication

Get all applications made by the authenticated tutor.

Example request:
curl --request GET \
    --get "https://rrn24.techchantier.com/tutor-finder/public/api/api/applications" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json"
const url = new URL(
    "https://rrn24.techchantier.com/tutor-finder/public/api/api/applications"
);

const headers = {
    "Content-Type": "application/json",

Register User

Create a new user account.

Example request:
curl --request POST \
    "https://rrn24.techchantier.com/tutor-finder/public/api/api/signup" \
    --header "Content-Type: multipart/form-data" \
    --header "Accept: application/json" \
    --form "name=aliquid"\
    --form "email=voconner@example.com"\
    --form "password=S;7nT*:C"\
    --form "phone_number=velit"\
    --form "user_type=dolores"\
    --form "location=enim"\
    --form "password_confirmation=laudantium"\
    --form "profile_image=@C:\Users\MD CHRISTIEN\AppData\Local\Temp\phpBFDA.tmp" \
    --form "profile_video=@C:\Users\MD CHRISTIEN\AppData\Local\Temp\phpBFDB.tmp" 
const url = new URL(
    "https://rrn24.techchantier.com/tutor-finder/public/api/api/signup"
);

const headers = {
    "Content-Type": "multipart/form-data",
    "Accept": "application/json",
};

const body = new FormData();
body.append('name', 'aliquid');
body.append('email', 'voconner@example.com');
body.append('password', 'S;7nT*:C');
body.append('phone_number', 'velit');
body.append('user_type', 'dolores');
body.append('location', 'enim');
body.append('password_confirmation', 'laudantium');
body.append('profile_image', document.querySelector('input[name="profile_image"]').files[0]);
body.append('profile_video', document.querySelector('input[name="profile_video"]').files[0]);

fetch(url, {
    method: "POST",
    headers,
    body,
}).then(response => response.json());

Accept

optional Video file for tutor profiles only (mp4, mov, avi) max 20MB Example: C:\Users\MD CHRISTIEN\AppData\Local\Temp\phpBFDB.tmp

Get a list of all available learning gigs. By default, pending gigs are excluded.

Example request:
curl --request GET \
    --get "https://rrn24.techchantier.com/tutor-finder/public/api/api/gigs?include=totam&search=vero&price=inventore&budget_period=sint&category_id=17&status=officia&learner_id=19&include_pending=1" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"search\": \"civrhcrashpoumrdmpikbcqwxvrewgyeciiygkcbpckvkrpkzrnllmzpzwiujwymwrw\",
    \"price\": \"47-94\",
    \"budget_period\": \"daily\",
    \"status\": \"in_progress\",
    \"include_pending\": false
}"

const url = new URL(
    "https://rrn24.techchantier.com/tutor-finder/public/api/api/gigs"
);

const params = {
    "include": "totam",
    "search": "vero",
    "price": "inventore",
    "budget_period": "sint",
    "category_id": "17",
    "status": "officia",
    "learner_id": "19",
    "include_pending": "1",
};
Object.keys(params)
    .forEach(key => url.searchParams.append(key, params[key]));

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "search": "civrhcrashpoumrdmpikbcqwxvrewgyeciiygkcbpckvkrpkzrnllmzpzwiujwymwrw",
    "price": "47-94",
    "budget_period": "daily",
    "status": "in_progress",
    "include_pending": false
};

fetch(url, {
    method: "GET",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

Request      

GET api/gigs

Headers

Content-Type      

Example: application/json

Accept      

Example: application/json

Query Parameters

include   string  optional  

Specify which relationships to include in the response. Multiple relationships can be comma-separated. Available relationships: learner, category, applications Example: totam

search   string  optional  

Search in gig title and description Example: vero

price   string  optional  

Filter by price range (format: 1000-5000) Example: inventore

budget_period   string  optional  

Filter by budget period (hourly/daily/weekly/monthly) Example: sint

category_id   integer  optional  

Filter gigs by category Example: 17

status   string  optional  

Filter by gig status (pending/open/in_progress/completed/cancelled) Example: officia

learner_id   integer  optional  

Filter gigs by the learner who created them Example: 19

include_pending   boolean  optional  

Include pending gigs (default: false) Example: true

Body Parameters

search   string  optional  

Must be at least 3 characters. Example: civrhcrashpoumrdmpikbcqwxvrewgyeciiygkcbpckvkrpkzrnllmzpzwiujwymwrw

price   string  optional  

Must match the regex /^\d+-\d+$/. Example: 47-94

budget_period   string  optional  

Example: daily

Must be one of:
  • hourly
  • daily
  • weekly
  • monthly
category_id   string  optional  

The id of an existing record in the categories table.

status   string  optional  

Example: in_progress

Must be one of:
  • pending
  • open
  • in_progress
  • completed
  • cancelled
learner_id   string  optional  

The id of an existing record in the users table.

include_pending   boolean  optional  

Example: false
